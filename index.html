HTML

  • Hypertext Markup Language (HTML) is one of the three cornerstone languages used in web development: HTML, CSS, and JS.
  • HTML is used to create the structure of a webpage.
  • HTML is made up of ELEMENTS and an element is a type of content on a webpage.
  • HTML elements work together to build the structure of a webpage and provide a framework that makes it easy for a user to navigate the webpage.
  • Two elements are especially important to understanding how the structure works: the HEAD and the BODY.
  • The HEAD element contains information, or metadata, that helps the browser to render the page correctly. Metadata is data about our webpage that the browser uses to effectively serve our webpage to users. Elements included within the HEAD are not visible when a user looks at a webpage.
  • The BODY element defines the body of the document, which contains the content that is visible to the user. All the elements that we want the user to see and interact with must be placed inside this element.
  • The BODY element is an example of a semantic HTML element. It is important to use semantic HTML elements, because they make your HTML easier to read by other developers and the browser and more accessible to a variety of users (assistive technologies).
  • An HTML ATTRIBUTE defines additional information about an element.
  • An element refers to the entire element from opening tag to closing tag, whereas a TAG refers only to what appears inside the angle brackets.
  • Elements that are nested inside other elements are called CHILD ELEMENTS. The element that holds all the children is known as the PARENT ELEMENT.
  • The ALT TEXT is important for screen readers and other assistive technologies. Alt text can also improve search engine rankings for your website.
  • SECTION elements are semantic elements that define a section in a document that contains thematic content.

CSS

  • Cascading Style Sheets (CSS) is used to add style to a webpage, it is a programming language used by developers to define how webpage content is presented to users.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • The most common way to style a webpage is by using an external CSS style sheet. This is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier (Separation of concerns).
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Every HTML element is essentially a box (CSS box model).
  • To apply a rule to all the elements visible on the page we use a special wild card symbol, or *.
  • We combined two selectors into a single CSS rule with a comma, allowing the declarations to affect both selectors.
  • Do Not Repeat Yourself (DRY) is a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained.
  • text-align is used as one way to center a title or heading on a webpage.
  • The IMG selector assigns property values to any image element in the markup.
  • To link an external CSS style sheet to an HTML file, you need to add the link to the style sheet in the HEAD section of the HTML file.
  • A LINK element creates the connection to the external file. The rel attribute specifies the relationship between the current document and the linked document resource. The href attribute specifies the location (URL) of the external resource.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.
  • The class attribute can be assigned to the element in the HTML, enabling us to freely choose which elements will be assigned the CSS rule.
